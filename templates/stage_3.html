Definitions

Abstraction - a way of managing a program's complexity by relying on multiple layers functions or "hidden functions." These hidden functions run behind the scenes in that they operate outside the view of the user. In Python and many other programming languages abstraction is dependent on prebuilt modules. These interact and exert direct influence on other compartmentalized aspects of the program. Abstraction simplifies programming, because it allows the programmer to focus on the task at hand rather than writing additional code.

Class - a prototype or generalized concept that is used to create objects. In other words, a class contains the attributes (both variables and methods) that describe an object. A class is often likened to a blueprint, although the analogy to a recipe works equally as well. For example, a recipe for a cake lists the ingredients and instructions on how to bake it. The recipe is a prototype for a cake, and from it many cakes can be made.

Recipe is to Class as Cake is to Object

Class Method - a function that is defined in a class and is therefore inherited by any object that is created from the class or any subclasses that are influenced by the superclass.

Class Variable - a variable (particular data type) that is defined in a class and is passed on to all objects created from it.

Constructor - also know as an initiation method (in Python written as __i__( )), it creates space in memory for the instance of a class.

Inheritance - the acquisition of attributes from a "parent" class. Classes can be arranged in a hierarchical fashion so that the higher class passes on attributes to anything below it, whether it be other classes or a single instance of that class. While this description is geared more towards Python programming, inheritance occurs elsewhere in the computer world. For example, in HTML attributes can be inherited due to nesting. If one <div> class is nested inside another <div> class the nested class will acquire the attributes assigned to the higher class. Instance - also known as an object. This can be thought of as the actualization of a class, or a unique copy of a class. An instance contains all of the attributes defined in the class model. Instance Method - a method that is assigned to an object (instance) when it is created and is defined in the class attributes.

Instance Variable - a variable that is assigned only to one particular object (instance) of a class.

Library - a collection of modules that can be drawn from in order to simplify the programming code. In addition to Python's Standard Library there are many other open-access libraries available.

Method - a function that belongs to and is defined by a class

Method Overriding - a process that takes place in the context of inheritance. In method overriding, a method (function) of a subclass (child class) takes precedent over the method of the superclass (parent class). In other words, the subclass overrides the superclass and disrupts inheriting its method.

Module - prewritten code that can itself be considered a partially executable program that becomes fully operational when imported into another program. The line that begins with import is used to do this.

Object - also known as an instance. This can be thought of as the actualization of a class, or a unique copy of a class. An instance contains all of the attributes defined in the class model. In our recipe-cake analogy, an instance is like a single cake created from the master recipe.

Lesson 2: A Closer Look at Object Oriented Programming
What is Object Oriented Programming, and why is it useful?

Object Oriented Programming (OOP) is a modular approach to programming that is based, appropriately enough, on objects. One of its hallmarks is its use and reuse of modules, a concept known as re-usability. Re-usability greatly increases the productiveness of a program in that he or she does not have rewrite the same code over and over again.

Another key aspect of OOP is that objects are arranged in hierarchies, thereby allowing for the condition known as inheritance. That is to say, OOP's design structure allows for the attributes of a higher element (class) to be imposed upon lower elements within the same grouping. Think of it like a good king ruling over his kingdom and passing along his goodness to his subjects who thereby also are good (that's a lot of goodness going on).

OOP also employs encapsulation, meaning that individual modules are dedicated to specific tasks and can interact with one another without necessarily affecting (altering) the contents of one another. This is sort of like a form of data protection and is known as information hiding.

With class-based programming languages like Python, objects are created from classes using methods known as constructors. In this sense, objects are instances of classes and are sometimes referred to as simply instances. Constructors make space in memory for the creation of an instance of a class.

So to summarize, OOP has these key features:

  • modular in nature
    • allows for re-use of pre-packaged code
    • makes programming faster and easier (in theory)
    • increases readability of code
  • arranged in hierarchies
    • allows for the condition known as inheritance
    • method overriding can be implemented so that a subclass retains its attributes (disrupts inheritance)
  • employs encapsulation
    • prevents individual modules from altering one another due to information hiding
    • part of what being modular means
